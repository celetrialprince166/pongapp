Cloud Map, in depth

service discovery

The frontend never knows a backend IP. It asks DNS for backend.pongapp.local every few seconds, and AWS Cloud Map answers with the live IPs of healthy backend tasks. Two halves make that work — registration (ECS puts tasks into Cloud Map) and resolution (nginx pulls them out).

modules/ecs-service/main.tfterraform

      

    

  

  

    
Who calls it?
The frontend nginx (proxy_pass http://$backend_upstream) for every /api, /ws, /media, /static request.

    
Where is it configured?
modules/cloudmap creates the namespace; modules/ecs-service creates the service + registers the backend via service_registries.

    
Why not an ALB?
East-west traffic stays internal — no per-LB hourly cost, lower latency, and ECS reports task health directly. The public ALB is reserved for the frontend edge.

Blue / green deploy path
ECS-native · CI/CD

  
A push to main runs the pipeline. The frontend uses ECS-native blue/green (deployment_configuration { strategy = "BLUE_GREEN" }): new tasks come up on a second target group, traffic shifts only after they pass health checks, then a bake window guards against regressions before the old version is torn down.
